# 要求仕様書（車載オートヘッドランプ制御）

## 1. 目的
車載ECU上で動作するオートヘッドランプ制御を実装し、AIエージェントが要求から設計・実装・テストまでを再現できるかを評価する。

## 2. システム境界
- 本ソフトはヘッドランプON/OFF判断のみを担当する。
- 実際のランプ駆動、CAN通信、OSスケジューラはシステム外。

## 3. 機能要求
- **FR-001**: 周囲照度が300 lux以下のサンプルが3回連続したらランプをONにする（デバウンス）。
- **FR-002**: 周囲照度が500 lux以上のサンプルが3回連続したらランプをOFFにする（ヒステリシス）。
- **FR-003**: 車速が10 km/h未満ではAUTOモードでランプをOFFとする。
- **FR-004**: トンネル検知時はAUTOモードで即時ランプをONにする。
- **FR-005**: 手動オーバーライド（FORCE_ON / FORCE_OFF / AUTO）を受け付ける。
- **FR-006**: FORCE_ON/FORCE_OFFは120秒でタイムアウトしAUTOに復帰する。
- **FR-007**: 照度センサ値が0未満または200000超の場合は異常フラグを立て、直前のランプ状態を維持する。
- **FR-008**: 起動直後（初回有効入力前）はランプをOFFとする。

## 4. 非機能要求
- **NFR-001**: C99準拠。
- **NFR-002**: 動的メモリを使用しない。
- **NFR-003**: 単体テストはGoogle Testで自動実行できること。

## 5. ベンチマークで特に評価したい「抜け漏れしやすい項目」
- デバウンス回数の境界（2回では切替しない）。
- 300/500 luxのヒステリシス（単一閾値で実装しない）。
- 手動オーバーライドのタイムアウト復帰。
- センサ異常時に状態遷移しないこと。
- 起動直後の安全側OFF。
