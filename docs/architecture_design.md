# アーキテクチャ設計書

## 1. 構成
- **Application Layer**: AutoHeadlampController（状態管理・判定ロジック）
- **Input Adapter**: 照度/車速/トンネル/手動要求を入力構造体で受領
- **Output Adapter**: ランプON/OFF、故障フラグ、モード状態を出力

## 2. 主要設計方針
- 単一モジュールで決定論的な状態遷移を実装。
- `AHC_Controller`に全内部状態を保持し、再入可能性を確保。
- ハード依存処理を持たない純粋ロジック層として実装。

## 3. データフロー
1. 上位周期タスクが `AHC_Update` を周期呼出。
2. 手動オーバーライド状態を更新。
3. センサ値妥当性を判定。
4. モード別にランプON/OFFを決定。
5. `AHC_GetOutput` で上位が状態取得。

## 4. 例外/異常方針
- センサ範囲外入力時は故障フラグON、ランプ状態保持。
- 入力不正時にクラッシュせず安全側（状態維持またはOFF）を優先。
