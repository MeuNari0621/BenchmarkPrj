# AIエージェント向けベンチマーク採点基準

合計100点（要求20 / アーキ20 / 設計20 / 実装20 / テスト20）

## 1. 要求仕様（20点）
- [ ] FR-001〜FR-008が識別可能なID付きで定義されている（8点）
- [ ] 非機能要求（C99、動的メモリ禁止、Google Test）が明記（4点）
- [ ] 抜け漏れしやすい項目が独立して明記（8点）

## 2. アーキテクチャ（20点）
- [ ] 層/責務分離が示される（6点）
- [ ] データフローと例外方針が明確（8点）
- [ ] ハード依存分離の方針が説明される（6点）

## 3. ソフト設計（20点）
- [ ] I/F、状態、判定順序が一貫して記述（8点）
- [ ] 要求IDと設計要素がトレーサブル（8点）
- [ ] 境界条件（閾値、回数、タイムアウト）記述（4点）

## 4. 実装（20点）
- [ ] FR-001〜FR-008を満たす（12点）
- [ ] 動的メモリ未使用、定数のマクロ化（4点）
- [ ] 異常入力時の安全動作（4点）

## 5. テスト（20点）
- [ ] 全FRに対応するテストが存在（8点）
- [ ] 抜け漏れしやすい項目を直接検証（8点）
- [ ] テストが自動実行可能（4点）

## 減点ルール（代表例）
- ヒステリシス未実装: -8点
- デバウンス不足（連続回数無視）: -8点
- 手動オーバーライドのタイムアウト未実装: -6点
- センサ異常で状態保持しない: -6点
- 起動時OFF未保証: -4点
