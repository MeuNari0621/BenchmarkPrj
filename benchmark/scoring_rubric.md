# AIエージェント向けベンチマーク採点基準

## 概要

本基準は、AIエージェントが要求仕様書のみを入力として、アダプティブクルーズコントロール（ACC）の設計・実装・テストを行った成果物を評価するためのものです。

## 配点構成（合計200点）

| カテゴリ | 配点 | 内容 |
|---------|------|------|
| 要求理解 | 30点 | 要求の正確な解釈と設計への反映 |
| アーキテクチャ | 25点 | モジュール分割、責務分離 |
| 詳細設計 | 25点 | インターフェース、状態遷移、アルゴリズム |
| 実装 | 60点 | 機能実現、コード品質 |
| テスト | 40点 | カバレッジ、境界値テスト |
| 非機能 | 20点 | C99準拠、動的メモリ禁止、堅牢性 |

## 減点ルール

### 重大な欠陥（各-20点）
- 緊急制動が動作しない
- フォルト時に制御が継続する
- 状態遷移がデッドロックする

### 中程度の欠陥（各-10点）
- 車速維持時に減速要求を出力する（FR-005違反）
- オーバーライド復帰が2秒より短い/長い
- 車間警告閾値の誤り

### 軽微な欠陥（各-5点）
- 単位変換の誤り（km/h⇔m/s）
- 境界条件の「以上」「未満」の誤り
- タイマー精度の誤差

### コンパイル/実行エラー
- コンパイル不可: 実装カテゴリ0点
- テスト失敗: 該当テスト項目0点

---

## 採点方法

1. **自動テスト実行**
   - 模範テストコードでビルド・実行
   - テスト通過率を計算

2. **静的解析**
   - 動的メモリ使用チェック
   - C99準拠チェック

3. **コードレビュー**
   - 設計書との整合性
   - コード品質の目視確認

4. **スコア計算**
   ```
   最終スコア = 基礎点 - 減点 + ボーナス点
   ボーナス: 全テスト通過 +10点
   ```
